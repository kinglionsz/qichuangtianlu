## ⚠️ 四、V2 待解决问题

### 4.1 🔴 高优先级 - 内联事件处理器（V1 问题 F，未修复）

**位置**: index.html 第36-38行, 第41行, 第97-99行

```html
<!-- 第36行 -->
<div class="hamburger" onclick="toggleMenu()">

<!-- 第41行 -->
<li><a href="#hero" onclick="closeMenu()">首页</a></li>

<!-- 第97-99行 -->
<button class="cyber-btn" id="btn-play" onclick="togglePlay()">PLAY</button>
<button class="cyber-btn" id="btn-speed" onclick="cycleSpeed()">1x</button>
<button class="cyber-btn" onclick="resetAnim()">RESET</button>
```

**风险**:
- ❌ 不符合现代前端最佳实践
- ❌ 无法批量解绑事件
- ❌ 内存泄漏风险
- ❌ 难以做事件委托

**建议修复方案**:

```javascript
// main.js 中统一绑定
document.addEventListener('DOMContentLoaded', () => {
  // 导航菜单
  document.querySelector('.hamburger')?.addEventListener('click', toggleMenu);
  document.querySelectorAll('.nav-links a').forEach(link => {
    link.addEventListener('click', closeMenu);
  });

  // 轨迹控制
  document.getElementById('btn-play')?.addEventListener('click', togglePlay);
  document.getElementById('btn-speed')?.addEventListener('click', cycleSpeed);
  document.querySelector('button:has-text("RESET")')?.addEventListener('click', resetAnim);
});
```

### 4.2 🟡 中优先级 - 全局变量污染（V1 问题 A，未修复）

**位置**: trajectory.js 第22-26行

```javascript
// ── 状态 ─────────────────────────────────────────────────────
let progress  = 0;
let playing   = false;  // 默认暂停，降低CPU占用
let speed     = 1;
let lastTime  = 0;
```

**风险**:
- ❌ 多个模块可能相互覆盖
- ❌ SSR/同构应用不友好
- ❌ 单元测试困难

**建议修复方案**:

```javascript
// 使用 IIFE 或 ES Module 作用域封装
const TrajectoryEngine = (() => {
  let _progress = 0;
  let _playing = false;
  let _speed = 1;
  let _lastTime = 0;

  return {
    getProgress: () => _progress,
    setProgress: (v) => { _progress = v; },
    isPlaying: () => _playing,
    togglePlay: () => { _playing = !_playing; },
    // ...
  };
})();
```

### 4.3 🟡 中优先级 - 缺少错误边界（V1 问题 E，未修复）

**位置**: ui.js 第21-24行

```javascript
export function openLightbox(el) {
  lightboxImg.src = el.querySelector('img').src;  // ⚠️ 可能抛出 NullReferenceError
  lightbox.classList.add('active');
}
```

**风险**:
- ❌ 如果 `el` 为 null 或没有 `img` 子元素，会导致 JS 错误
- ❌ 页面可能卡在不可用状态

**建议修复**:

```javascript
export function openLightbox(el) {
  const img = el?.querySelector('img');
  if (!img || !lightboxImg) {
    console.warn('[UI] Lightbox: No image found in element');
    return;
  }
  lightboxImg.src = img.src;
  lightbox.classList.add('active');
}
```

### 4.4 🟢 低优先级 - Canvas 尺寸硬编码（V1 问题 C，未修复）

**位置**: trajectory.js 第13-14行

```javascript
const CW = canvas.width;   // 1100
const CH = canvas.height;  // 770
```

**说明**:
- 目前 Canvas 尺寸与 HTML 中 `width="1100" height="770"` 保持一致
- 但缺乏响应式适配

**建议**: 使用 ResizeObserver 动态监听

```javascript
const resizeObserver = new ResizeObserver((entries) => {
  for (const entry of entries) {
    const { width, height } = entry.contentRect;
    canvas.width = width;
    canvas.height = height;
  }
});
resizeObserver.observe(canvas.parentElement);
```
