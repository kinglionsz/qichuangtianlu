## Codebase Overview

This is a single-page HTML application that visualizes the 2017 骑闯天路 (Ride the Heavenly Road) cycling event in Shenzhen. The page features a cyberpunk/synthwave aesthetic with neon colors, glitch effects, and animated cycling trajectory.

## Key Components

- `index.html`: The main file containing all HTML, CSS, and JavaScript
- `bike.jpg`: Background image for the hero section
- `route-map.jpg`: Route map image displayed in the gallery
- `winner1.jpg`, `winner2.jpg`: Event photos displayed in the gallery

## Architecture and Structure

The application is a self-contained single-page app with:

1. **HTML Structure**: Semantic sections for hero, trajectory, route checkpoints, details, and gallery
2. **CSS Styling**: Extensive use of CSS variables for the neon color scheme, custom properties for animations, and responsive design with media queries
3. **JavaScript Functionality**: Canvas-based trajectory visualization with:
   - Dynamic cycling progress animation
   - Real-time telemetry display (distance, elevation, speed, time)
   - Elevation profile chart
   - Route segment labeling
   - Interactive controls (play/pause, speed adjustment, reset)

## Development Commands

This project uses Vite as a development server and build tool. The available commands are:

- `npm run dev`: Start the development server with hot reloading at http://localhost:5173
- `npm run build`: Build the project for production in the `dist` directory
- `npm run preview`: Preview the production build locally

To develop:

1. Run `npm run dev` to start the development server
2. Edit the source files in the `src` directory
3. Changes are automatically reloaded in the browser
4. When ready to deploy, run `npm run build`

## Key Files and Locations

- `src/index.html`: Main HTML file
- `src/styles/main.css`: Main CSS file with variables and styling
- `src/js/main.js`: Entry point for JavaScript
- `src/js/trajectory.js`: Handles the canvas rendering and animation loop
- `src/js/trajectoryData.js`: Contains all route waypoints and metadata
- `src/js/ui.js`: Handles UI interactions (lightbox, mobile menu, scroll reveal)
- `#trajectory-canvas`: Canvas element for the cycling trajectory visualization
- `wp` array: Contains all route waypoints with Canvas coordinates and metadata
- `route` variable: Reference to the `wp` array used by the visualization
- `progress` variable: Controls the animation progress (0.0 to 1.0)

## Project Structure

The project follows a modular structure with separate files for different concerns:

- `src/`: Source code directory
  - `assets/`: Static assets (images)
  - `js/`: JavaScript modules
    - `main.js`: Module entry point
    - `trajectory.js`: Canvas rendering and animation
    - `trajectoryData.js`: Route data and constants
    - `ui.js`: User interface interactions
  - `styles/`: CSS stylesheets
    - `main.css`: Main stylesheet with variables and component styles

## Key Technical Details

- The trajectory visualization uses direct Canvas pixel coordinates rather than GPS-to-Canvas mapping for accuracy
- Route data is hardcoded in the `wp` array with Canvas coordinates and GPS metadata
- Animation loop runs at approximately 30fps using `requestAnimationFrame` with frame rate throttling
- Performance optimizations include pulse value caching (updated every 10 frames)
- Responsive design with mobile-friendly layout
- Scroll-based reveal animations using IntersectionObserver
- The application uses ES6 modules with Vite for development and bundling

## Important Implementation Notes

- The route visualization uses direct Canvas pixel coordinates defined in the `coastPts` and `wp` arrays
- GPS coordinates are stored as metadata for display but are not used for positioning
- The peninsula map orientation is from southwest (left) to northeast (right)
- Animation speed is controlled by the `speed` variable and `cycleSpeed()` function
- The `posAt(t)` function interpolates between waypoints for smooth animation
- The `drawElevation()` function renders the elevation profile chart at the bottom of the canvas
- The `drawCheckpoints()` function renders the checkpoint markers with pulsing animations
- The `drawHUD()` function renders the telemetry overlay with current stats
- The `drawBike()` function renders the animated cyclist at the current position
- The `frame()` function is the main animation loop that updates and renders the entire visualization
- All UI controls are exported from `trajectory.js` and `ui.js` and attached to the `window` object in `main.js` for use in HTML inline event handlers

## 关键技术细节

- 轨迹可视化使用直接的 Canvas 像素坐标，而非 GPS 到 Canvas 映射，以确保准确性
- 路线数据在 `wp` 数组中硬编码，包含 Canvas 坐标和 GPS 元数据
- 动画循环使用 `requestAnimationFrame` 以大约 30fps 运行，并带有帧率节流
- 性能优化包括脉冲值缓存（每 10 帧更新一次）
- 响应式设计，支持移动设备友好布局
- 基于滚动的揭示动画使用 IntersectionObserver
- 应用使用 ES6 模块和 Vite 进行开发和打包

## 重要实现说明

- 路线可视化使用在 `coastPts` 和 `wp` 数组中定义的直接 Canvas 像素坐标
- GPS 坐标作为元数据存储用于显示，但不用于定位
- 半岛地图方向为从西南（左）到东北（右）
- 动画速度由 `speed` 变量和 `cycleSpeed()` 函数控制
- `posAt(t)` 函数在航点之间插值以实现平滑动画
- `drawElevation()` 函数在 Canvas 底部渲染海拔剖面图
- `drawCheckpoints()` 函数渲染带有脉冲动画的检查点标记
- `drawHUD()` 函数渲染带有当前状态的遥测叠加层
- `drawBike()` 函数在当前位置渲染动画骑行者
- `frame()` 函数是主动画循环，更新并渲染整个可视化
- 所有 UI 控件从 `trajectory.js` 和 `ui.js` 导出，并在 `main.js` 中附加到 `window` 对象，用于 HTML 内联事件处理器
