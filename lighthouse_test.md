🔴 严重问题

1. HTML 中 toggleMenu / closeMenu 全局函数时机问题
index.js 用了 IIFE 包裹，虽然最后通过 window.xxx 暴露了全局函数，但 HTML 的 onclick 绑定在 DOM 上时 JS 可能还没执行完。如果脚本加载延迟，点击汉堡菜单会报 Uncaught ReferenceError。


建议：HTML 中移除 onclick，改用 JS 内事件委托。


2. CSS 选择器拼写不匹配
CSS 中有 .winner-card.first / .winner-card.second / .winner-card.third 和 .winner-card.winner-champion / .winner-card.winner-second / .winner-card.winner-third 两套类名

.winner-rank 这个类名在 HTML 中根本不存在（HTML 用的是 .winner-badge），所以 CSS 中 .winner-card.first .winner-rank 这些规则完全无效

3. 图片路径硬编码为 /imgs/xxx
CSS 中 hero-bg 使用 url(/imgs/qichuangtianlu-...jpg)，HTML 中 <source srcset="/imgs/xxx.webp"> —— 这些路径依赖部署环境根目录下有 /imgs/ 文件夹。如果直接本地打开或部署到子路径，所有图片都无法加载。


建议：改为相对路径 ./imgs/ 或用实际可用的 URL。

HTML 中图集区块用的是 .routebook-gallery，但 CSS 定义了 .gallery-section（不匹配），图集的 grid 布局实际靠 .routebook-gallery .gallery-grid 继承样式——但这依赖级联，不够稳健。
